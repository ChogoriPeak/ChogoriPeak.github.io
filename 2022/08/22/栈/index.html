判断是否为符合规则的括号：

给定一个只包括 ‘(‘，’)’，’{‘，’}’，’[‘，’]’ 的字符串 s ，判断字符串是否有效。

有效字符串需满足：

左括号必须用相同类型的右括号闭合。
左括号必须以正确的顺序闭合。

这里主要用了字典和栈的定义，这里的栈不能是空的，因为判断右括号的时候，如果第一个就是又括号的话会报错。这里是判断是不是左括号，如果不是的话，则弹出之前储存的左括号进行判断，成对的话即走下一轮。

设计一个支持 push ，pop ，top 操作，并能在常数时间内检索到最小元素的栈。

实现 MinStack 类，包含以下内容：

MinStack() 初始化堆栈对象。
void push(int val) 将元素val推入堆栈。
void pop() 删除堆栈顶部的元素。
int top() 获取堆栈顶部的元素。
int getMin() 获取堆栈中的最小元素。

这个很容易理解，只要多设计一条栈用来储存最小值即可，如果最小值被推出，则第二小的值会顶替上去。其他没什么难度。

每日温度——单调栈：

给定一个整数数组 temperatures ，表示每天的温度，返回一个数组 answer ，其中 answer[i] 是指对于第 i 天，下一个更高温度出现在几天后。如果气温在这之后都不会升高，请在该位置用 0 来代替。如：

代码思路如下：这里采用了单调栈的方式，我们设定一个栈，这个栈储存符合条件的气温的下标，这里的符合条件指的是：最高气温下标放下面，低的放上面，但如果又比栈内气温大的则需要全部将低于其的气温下标出栈，并返回该下标，当前气温日减去该下标的值就等于下一个更高温度的天数。最后返回这个对比表。

简单来说，就是计算每一天还有多少天后会比现在更热然后存储成表格。
