这里我们可以简化成1->2->3需要变成3->2->1的情况，此时可以看出只需要将链表之间的链接关系互换即可，这里我们采用双指针的方式，其中一个是head结点的cur指针，另一个是cur之前的指针pre，指向Null。

这里的算法循环是当cur运行到Null的时候，返回的pre刚好为新链表的head指针。

先用temp指针保存一下cur的下一个节点地址，然后将cur与pre链接反向，此时pre向前移动一位指向cur，cur再移动一位到达原来的temp地址

ps：此时pre与之前的反向链接，而cur照常与正向的链接。

相交链表:

当两个链表有相交点的时候，此时总存在有一个公倍数使得两个链表之间同时遍历到同一个指针上，这也就是为什么A和B之间遍历的时候到达Null就互相转换head节点的原因。

当两个链表没有相交点的时候，总会有一个时刻两者都到达Null，此时两者相等，返回的值为Null，符合题意。、

合并有序链表：

当两个链表都是有序的时候。此时合并的话需要考虑两种情况，一种是常规的对比，一种是有链表到达尽头时的链表的指向。该算法的重点在于哨兵节点的定义，没有这个哨兵节点的话，很难返回新定义的链表的头节点。

循环结束条件：有一个链表执行到最后，出现Null的时候跳出循环。当跳出循环的时候，如果某个链表仍存在剩余的节点，则直接将后续的节点接入新的链表结尾。

两数之和：

class Solution:
    def twoSum(self, nums: List[int], target: int) -> List[int]:
        #定义一个哈希表
        hash = {}
        #将没有读取过的值存进哈希表
        for i, num in enumerate(nums):
            comple = target - num
            if comple not in hash:
                hash[num] = i;
            else:
                return [hash[comple], i]
        return 0

这里我第一次系统学习了哈希表，哈希表与数组、链表不同，主要可以通过值来查找序号，十分方便。

倒数第k个链表

输入一个链表，输出该链表中倒数第k个节点。为了符合大多数人的习惯，本题从1开始计数，即链表的尾节点是倒数第1个节点。

这里有两种方法，一种是遍历该链表并获取其长度，通过n-k的方法取得倒数k值。

另外的方法是使用快慢指针，其中快指针领先慢指针k个节点，当快指针移动到Null的时候，慢指针刚好移动至倒数k个节点上。
