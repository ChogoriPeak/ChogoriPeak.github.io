用栈实现队列：

请你仅使用两个栈实现先入先出队列。队列应当支持一般队列支持的所有操作（push、pop、peek、empty）：

实现 MyQueue 类：

void push(int x) 将元素 x 推到队列的末尾
int pop() 从队列的开头移除并返回元素
int peek() 返回队列开头的元素
boolean empty() 如果队列为空，返回 true ；否则，返回 false

这里的主要思想是借助辅助栈，当我们1栈弹出时，输入到2栈，此时最先进的数据会被放在2栈的顶端第一个输出，从而实现FIFO.

而推入操作则直接往1栈入栈，弹出操作稍微比较麻烦点，主要是需要判断2栈是否为空，如果为空，则需要将1栈元素全数装进2栈，再弹出，如果非空，则直接弹出2栈顶部元素即可。

获取第一个元素的操作比较复杂，这里主要套用了弹出的函数，并获取该弹出数值，最后将该元素塞回2栈。一个巧妙的思路。而检查非空的函数，只需要检查1栈和2栈是否为空即可。

循环队列：

循环队列的一个好处是我们可以利用这个队列之前用过的空间。在一个普通队列里，一旦一个队列满了，我们就不能插入下一个元素，即使在队列前面仍有空间。但是使用循环队列，我们能使用这些空间去存储新的值。

你的实现应该支持如下操作：

MyCircularQueue(k): 构造器，设置队列长度为 k 。
Front: 从队首获取元素。如果队列为空，返回 -1 。
Rear: 获取队尾元素。如果队列为空，返回 -1 。
enQueue(value): 向循环队列插入一个元素。如果成功插入则返回真。
deQueue(): 从循环队列中删除一个元素。如果成功删除则返回真。
isEmpty(): 检查循环队列是否为空。
isFull(): 检查循环队列是否已满。

实现该类的方法主要采用了比较麻烦的函数，首先是队列需要定义两个指针，一个指向队头一个指向队尾，而队尾元素一般是空，即用来存放新元素的，此时需要K+1个空间，这里的K指的是队列元素个数。

入队的话一般需要判断该队列是否为满，可以直接借用类内方法。另外如果队列未满，则需要将值输入rear元素所指位置，并rear指针向后移动一位。因为这里循环的原因，队列前面的空间也可以使用，所以为了不浪费空间，这里rear采用的是(self.rear + 1) % len(self.element)，这里能够有效地帮助我们返回前面未使用的空间。

出队同样需要判断队列是否为空，同样借用类内方法，另外，与入队类似需要将推出的数值置零，并将队头指针向后移动一位，即采用(self.front + 1) % len(self.element).

返回队头队尾元素，则需要判空，否则执行对应的操作即可，返回队头元素不需要考虑循环，但是返回队尾元素需要。

判空的要求即是队头指针与队尾指针相同，而判满则是队尾的下一个元素就是队头。
